Прекрасно, переходим к модулю квестов (`DayZ-Expansion-Quests`). Эта система довольно объёмная и связывает между собой НПС, задачи (цели) и глобальные настройки перезапуска.

Как и в прошлый раз, переводим описания максимально адаптивно для администраторов и ИИ, сохраняя технические параметры, типы данных и структуру JSON в идеальном Markdown.

---

```markdown
# Документация по настройке DayZ Expansion Quests (Квесты)

## 1. Общие принципы работы квестов

> 🛠️ **Внимание:** Раздел находится в стадии наполнения разработчиками, возможны изменения в будущих обновлениях.

Для корректной работы и запуска любого квеста системе необходима связка из нескольких обязательных компонентов:
1. **Квестовые НПС (NPC):** Персонажи или объекты, которые выдают квест или принимают его обратно (если квест не завершается автоматически).
2. **Задачи / Цели квеста (Objectives):** Конкретные условия, которые игроку или группе необходимо выполнить, чтобы продвинуться по сюжету или завершить квест.
3. **Конфигурационный файл самого квеста:** Объединяет НПС, цели, текстовые диалоги, требования и награды в единое целое.

Квест будет отображаться в меню взаимодействия и станет доступен для взятия только тогда, когда полностью собраны все его файлы конфигурации и игрок удовлетворяет всем выставленным условиям. Если НПС отказывается выдавать квест — ищите ошибку в зависимостях или синтаксисе JSON.

## 2. Quest Configuration (Конфигурация квеста)

Файлы квестов можно называть как угодно, главное — использовать расширение `.json` и размещать их в папке конфигурации квестов. Система подгрузит их автоматически.

### Параметры конфигурации квеста

| Параметр | Тип данных | Описание |
| :--- | :--- | :--- |
| `"ConfigVersion"` | Integer | Текущая версия файла конфигурации. Не изменяйте её без необходимости. |
| `"ID"` | Integer | **Уникальный ID квеста**. Следите, чтобы этот ID не повторялся ни в одном другом файле квестов на сервере. |
| `"Type"` | Integer | Тип квеста из перечисления `ExpansionQuestType`. На данный момент для всех квестов используется значение `1` (`NORMAL`). |
| `"Title"` | String | Название (заголовок) квеста, отображаемое в интерфейсе. |
| `"ObjectiveText"` | String | Короткий понятный текст-описание задачи (например: *«Убить 10 зомби»*). |
| `"FollowUpQuest"` | Integer | ID следующего (последующего) квеста. Если квесты идут цепочкой, укажите здесь ID следующего задания, и оно автоматически выдастся игроку при сдаче текущего. |
| `"Repeatable"` | Boolean | Повторяемый квест (`true`/`false`). Разрешает проходить задание заново. |
| `"IsDailyQuest"` | Boolean | Ежедневный квест. Имеет кулдаун до общесерверного времени сброса ежедневок. |
| `"IsWeeklyQuest"` | Boolean | Еженедельный квест. Имеет кулдаун до общесерверного времени сброса еженедельных заданий. |
| `"CancelQuestOnPlayerDeath"` | Boolean | Отмена при смерти. Квест автоматически сбросится/отменится, если выживший (или любой член его отряда, если квест групповой) погибнет. |
| `"Autocomplete"` | Boolean | Автозавершение. Квест мгновенно зачтется и выдаст награду в полевых условиях, как только будут выполнены все задачи (идти к НПС за сдачей не нужно). |
| `"IsGroupQuest"` | Boolean | Групповой квест (`true`/`false`). Прогресс квеста засчитывается на весь отряд. |
| `"ObjectSetFileName"` | String | Скриптовый спавн объектов. Имя `.map` файла (без расширения) из папки `MISSION.MAPNAME/expansion/quests/objects`. Позволяет заспавнить кастомные декорации, ящики или преграды в мире строго на время выполнения игроком этого квеста. |
| `"NeedToSelectReward"` | Boolean | Выбор награды игроком:<br>`0` — Игрок получает абсолютно все предметы, прописанные в массиве наград.<br>`1` — Из списка предложенных наград игрок обязан сам выбрать только один предмет при сдаче квеста. |
| `"RandomReward"` | Boolean | Случайная награда. Если включено, награды будут выдаваться рандомно на основе индивидуального шанса (`Chance`) предметов. |
| `"RandomRewardAmount"` | Integer | Количество случайных предметов, которое получит каждый игрок при активированном параметре `RandomReward`. |
| `"RewardsForGroupOwnerOnly"` | Boolean | Награда только лидеру группы. Если квест групповой, при значении `true` награду получит только создатель/владелец отряда. |
| `"QuestGiverIDs"` | Array | Массив уникальных ID квестовых НПС, которые могут **выдать** этот квест. |
| `"QuestTurnInIDs"` | Array | Массив уникальных ID квестовых НПС, которым нужно **сдать** выполненный квест. |
| `"IsAchievement"` | Boolean | Достижение (Ачивка). Квест автоматически и скрытно выдается всем новым игрокам при первом заходе на сервер. |
| `"QuestColor"` | Integer | Основной цвет квеста для кастомизации его отображения в меню и HUD-виджете. |
| `"ReputationReward"` | Integer | *Требуется Hardline.* Количество очков репутации, получаемое игроком в награду за выполнение квеста. |
| `"ReputationRequirement"` | Integer | *Требуется Hardline.* Минимальное количество очков репутации, необходимое игроку, чтобы квест вообще отобразился у НПС и его можно было взять. |
| `"PreQuestIDs"` | Array | Массив ID предварительных квестов. Список заданий, которые игрок **обязан завершить ранее**, чтобы открыть доступ к текущему квесту. |
| `"RequiredFaction"` | String | *Требуется модуль AI.* Название фракции ботов, в которой должен состоять игрок, чтобы взять квест. |
| `"FactionReward"` | String | *Требуется модуль AI.* Название фракции ботов, в которую игрок автоматически перейдет в качестве награды за выполнение. |
| `"PlayerNeedQuestItems"` | Boolean | Проверка квестовых вещей при реконнекте. Если у игрока при перезаходе на сервер в инвентаре не окажется выданного квестового предмета, квест автоматически отменится. |
| `"DeleteQuestItems"` | Boolean | Удалять ли квестовые предметы (инструменты/ключи, выданные на время квеста) из инвентаря игрока при успешном завершении задания (`true`/`false`). При отмене квеста они удаляются всегда. |
| `"SequentialObjectives"` | Boolean | Последовательные задачи:<br>`0` — Все задачи квеста активны одновременно, их можно выполнять в любом порядке.<br>`1` — Строгая последовательность. Вторая задача не начнет засчитываться, пока игрок полностью не выполнит первую. |
| `"SuppressQuestLogOnCompletion"` | Boolean | Скрывать лог квеста в книге при автозавершении, если у квеста не привязаны `QuestGiverIDs`. |
| `"Active"` | Boolean | Активация файла конфигурации (`0` — выключен и не загружается сервером, `1` — включен). |

---

### Текстовые диалоги (`"Descriptions"`)
Массив строк, который должен содержать **строго до 3-х записей** (по одной строке на каждый этап диалога с НПС):
1.  *Строка 1:* Текст при взятии квеста (постановка задачи НПС).
2.  *Строка 2:* Текст НПС, если игрок возвращается к нему с еще не выполненным активным квестом.
3.  *Строка 3:* Текст НПС при успешной сдаче и получении награды.

---

### Специальные структуры данных квеста

#### Временные квестовые предметы (`"QuestItems"`)
Предметы, выдаваемые на время квеста. Они защищены от абуза: удаляются при отмене задания, выходе игрока или рестарте, и выдаются обратно в инвентарь при реконнекте, пока квест активен.
```json
"QuestItems": [
    { "ClassName": "SledgeHammer", "Amount": 1 }
]

```

#### Награды (`"Rewards"`)

```json
"Rewards": [
    {
        "ClassName": "TaloonBag_Blue", // Класснейм награды
        "Amount": 1,                    // Количество предметов
        "Attachments": [],              // Список модулей/лута внутри предмета
        "DamagePercent": 0,             // % износа/урона предмета и его содержимого
        "QuestID": -1,                  // Если > 0, предмет станет "квест-гивером" (активирует квест при клике на него)
        "Chance": 1.0                   // Шанс выпадения (для RandomReward)
    }
]

```

#### Цели в конфиге квеста (`"Objectives"`)

Массив, связывающий квест с файлами задач из папки целей:

```json
"Objectives": [
    { "ConfigVersion": 21, "ID": 3, "ObjectiveType": 3 }
]

```

#### Фракционная репутация (`"FactionReputationRequirements"` / `"FactionReputationRewards"`)

Объекты формата `Map<String, Integer>` для регулирования отношений с фракциями eAI:

```json
"FactionReputationRequirements": { "Survivors": 500 }, // Нужно 500 репутации у Выживших
"FactionReputationRewards": { "Survivors": 100 }       // Награда: +100 репутации

```

### Шаблоны особых типов квестов

#### 1. Авто-старт квест (Выдается автоматически при заходе на сервер)

Для корректной работы авто-старта необходимо строго соблюсти следующие параметры:

* `"QuestGiverIDs": []` — Массив выдающих НПС должен быть **пустым**!
* `"IsAchievement": 0` — Флаг достижения должен быть выключен!
* `"IsGroupQuest": 0` — Не должен быть групповым!
* `"PreQuestIDs": []` — Не должен иметь предварительных условий!

#### 2. Квест-Достижение (Скрытая ачивка)

Выдается игроку незаметно при первом коннекте, никак не отображается в HUD и не шлет уведомлений, пока игрок его полностью не выполнит.

* `"QuestGiverIDs": []` — Обязательно пустой массив НПС.
* `"IsAchievement": 1` — Флаг достижения включен.
* `"Autocomplete": 1` — Квест обязан завершаться автоматически прямо в поле.
* `"IsGroupQuest": 0`, `"PreQuestIDs": []` — Пустые зависимости.

#### 3. Ежедневные и Еженедельные квесты

* **Daily:** `"Repeatable": 1`, `"IsDailyQuest": 1`, `"IsWeeklyQuest": 0`
* **Weekly:** `"Repeatable": 1`, `"IsDailyQuest": 0`, `"IsWeeklyQuest": 1`

## 3. Quest Objectives Configuration (Конфигурация целей квеста)

Файлы задач лежат в отдельных папках в зависимости от типа цели. Каждый файл настраивает одно конкретное действие.

### Параметры конфигурации целей

* **`"ConfigVersion"`** (Integer) — Версия конфига целей. Не изменять.
* **`"ID"`** (Integer) — **Уникальный ID цели внутри своей категории**. Вы можете использовать ID `1` для цели типа TARGET и ID `1` для цели типа TRAVEL, но нельзя создавать две цели TARGET с одинаковым ID `1`.
* **`"ObjectiveType"`** (Integer) — Цифровой идентификатор типа цели. Указывает системе, по какому алгоритму обрабатывать задачу:
* `2` = **TARGET** — Убить определенное число зараженных, животных или игроков. *(Опционально: из конкретного оружия)*.
* `3` = **TRAVEL** — Добраться до точки с координатами X, Y, Z.
* `4` = **COLLECT** — Найти/собрать определенное количество предметов.
* `5` = **DELIVERY** — Доставить указанные вещи НПС-приемщику в конкретную точку.
* `6` = **TREASUREHUNT** — Поиск сокровищ (найти зарытый/спрятанный тайник по наводке).
* `7` = **AIPATROL** — Ликвидировать патруль вооруженных ИИ-ботов.
* `8` = **AICAMP** — Штурмовать и полностью зачистить укрепленный лагерь ИИ-ботов.
* `9` = **AIVIP (ESCORT)** — Защитить и сопроводить квестового ИИ-НПС до безопасной точки маршрута.
* `10` = **ACTION** — Выполнить определенное контекстное действие (скриптовое).
* `11` = **CRAFTING** — Скрафтить (создать) определенные предметы по рецепту.


* **`"ObjectiveText"`** (String) — Текст задачи, выводимый в лог книги и HUD-виджет.
* **`"TimeLimit"`** (Integer) — Ограничение по времени в секундах, выделяемое на выполнение этой цели.
* **`"Active"`** (`true`/`false`) — Статус загрузки файла системой целей.

## 4. Quest NPC Configuration (Конфигурация квестовых НПС)

Этот файл спавнит НПС или статический объект (доску объявлений) в мире, настраивает его внешность, фракцию, анимации и поведение.

### Параметры конфигурации НПС

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"ConfigVersion"` | Integer | Текущая версия файла конфигурации НПС. |
| `"ID"` | Integer | **Уникальный ID НПС**. Используется для привязки квестов в параметрах `QuestGiverIDs` / `QuestTurnInIDs`. |
| `"ClassName"` | String | Класснейм спавнящейся сущности. Мод предоставляет огромный список готовых пресетов внешности (см. ниже). |
| `"Position"` / `"Orientation"` | Vector | Координаты `[X, Y, Z]` спавна НПС в мире после старта сервера и направление его взгляда. |
| `"NPCName"` | String | Имя персонажа, отображаемое при наведении курсора и в меню действий. |
| `"DefaultNPCText"` | String | Дефолтная реплика НПС, если у игрока нет доступных квестов у этого персонажа (например: *«Хмм?»*, *«У меня для тебя ничего нет»*). |
| `"Waypoints"` | Array | Массив координат маршрута. Используется только для ИИ-типов при наличии мода *eAI*. Первой точкой обязательно должна быть координата спавна! Если точек больше одной, НПС будет циклически патрулировать по вейпоинтам. |
| `"NPCEmoteID"` / `"NPCInteractionEmoteID"` | Integer | ID анимации (эмоции), которую НПС крутит в простое и при открытии диалога с ним (см. справочник ID ниже). |
| `"NPCEmoteIsStatic"` | Boolean | Зафиксировать анимацию (`0`/`1`). Если `1`, НПС намертво застынет в одной указанной анимации и никогда её не сменит (удобно для сидячих НПС или часовых). |
| `"NPCLoadoutFile"` | String | Название файла экипировки (лоадаута) из папки листов снаряжения Expansion. Задает одежду и оружие НПС. |
| `"NPCQuestCancelEmoteID"` | Integer | ID анимации НПС при отмене игроком квеста (например, пожимание плечами). |
| `"NPCQuestStartEmoteID"` | Integer | ID анимации НПС при успешном взятии квеста игроком (например, кивок). |
| `"NPCQuestCompleteEmoteID"` | Integer | ID анимации НПС при успешной сдаче квеста (например, аплодисменты). |
| `"NPCFaction"` | String | *Для eAI.* Фракция, к которой принадлежит данный НПС (влияет на его агрессию к другим фракциям и игрокам). |
| `"NPCType"` | Integer | Тип сущности для корректной инициализации спавн-ивента:<br>

<br>`0` = **NORMAL** (Обычный статичный НПС-человек).<br>

<br>`1` = **OBJECT** (Статический объект карты — шкаф, доска).<br>

<br>`2` = **AI** (Полноценный eAI-бот, способный ходить и стрелять). |

---

### Справочники класснеймов и эмоций НПС

#### Доступные типы объектов (для `"NPCType": 1`):

Если вы хотите сделать выдачу квестов через доску объявлений или шкафчик, пропишите в `"ClassName"` один из этих классов:

* `ExpansionQuestObjectBoard` (Квестовая доска)
* `ExpansionQuestBoardSmall` (Малая доска)
* `ExpansionQuestBoardLarge` (Большая доска объявлений)
* `ExpansionQuestObjectLocker` (Квестовый шкафчик)

*Прим: Для стандартных НПС-людей используются пресеты вида `ExpansionQuestNPCMirek`, `ExpansionQuestNPCMaria` или их eAI-версии с приставкой `AI` (`ExpansionQuestNPCAIMirek`), если загружен Expansion-AI.*

#### Справочник ID анимаций (`NPCEmoteID`):

* `1` — GREETING (Приветствие)
* `4` — TAUNT (Провокация/Насмешка)
* `5` — LYINGDOWN (Лечь на землю)
* `7` — FACEPALM (Рука-лицо)
* `11` — SUICIDE (Суицид)
* `12` — DANCE (Танец)
* `14` / `15` — SIT A / SIT B (Сесть на землю)
* `39` — CLAP (Аплодисменты)
* `44` — SALUTE (Воинское приветствие)
* `46` — WATCHING (Осматриваться)
* `58` — NOD (Кивок согласия)
* `59` — SHAKE (Отрицательное качание головой)
* `60` — SHRUG (Пожимание плечами)
* `61` — SURRENDER (Сдаться/Руки за голову)

## 5. Quest Settings (Глобальные настройки системы квестов)

Этот файл управляет поведением всей квестовой экосистемы на сервере, текстами системных нотификаций, лимитами и временем ресета.
Находится по пути: `mpmissions\dayzOffline.<имя_карты>\expansion\settings\QuestSettings.json`

### Основные параметры системы

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"m_Version"` | Integer | Текущая версия файла общих настроек. Не изменять. |
| `"EnableQuests"` | Boolean | Общий рубильник системы квестов на сервере (`0` — выключено, `1` — включено). |
| `"EnableQuestLogTab"` | Boolean | Интеграция с Книгой. Добавляет вкладку текущих квестов (Квест-лог) в инфо-книгу на клавишу **B** (требуется *Expansion-Book*). |
| `"CreateQuestNPCMarkers"` | Boolean | Авто-маркеры НПС на карте навигации сервера. *В текущих версиях фича находится на доработке разработчиками мода.* |
| `"MaxActiveQuests"` | Integer | Максимальное количество квестов, которое один игрок может держать активными (взятыми) одновременно. Например: `5`. |
| `"UseQuestNPCIndicators"` | Boolean | Отображение 3D-индикаторов (восклицательных/вопросительных знаков) над головами НПС в мире, если у них есть доступные или готовые к сдаче квесты. |

---

### Групповые режимы выполнения (`"GroupQuestMode"`)

Определяет права участников отряда при прохождении групповых квестов:

* **`0`** — Строгий режим. Только создатель/лидер группы (Group Owner) имеет право брать у НПС и сдавать групповые квесты.
* **`1`** — Смешанный режим. Взять групповой квест у НПС может абсолютно любой член отряда, но сдать выполненное задание и получить награду может только лидер группы.
* **`2`** — Демократичный режим. Любой участник группы имеет полные права как на взятие квестов у НПС, так и на их финальную сдачу от лица всего отряда.

---

### Тайминги сброса циклических заданий

* **`"UseUTCTime"`** (`0`/`1`) — Использовать международное скоординированное время UTC для расчета кулдаунов и перезапусков квестов. При значении `0` расчет идет по локальному времени машины, на которой запущен сервер.
* **`"DailyQuestResetHour"`** / **`"DailyQuestResetMinute"`** (Integer) — Время (час и минуты) проведения ежесуточного автоматического сброса и перезапуска ежедневных квестов (например, `8` и `0` — сброс каждый день в 08:00 утра).
* **`"WeeklyQuestResetDay"`** (String) — День недели на английском языке, в который происходит глобальный сброс еженедельных квестов (например, `"Wednesday"` — среда).
* **`"WeeklyQuestResetHour"`** / `"WeeklyQuestResetMinute"` (Integer) — Точное время сброса еженедельных квестов в указанный выше день недели.

---

### Тексты системных всплывающих уведомлений

Нижеуказанные параметры настраивают заголовки (`Title`) и внутреннее текстовое содержание (`Text`) всплывающих окон, информирующих игрока об изменении статуса квеста. Переменная `%1` подставляет имя квеста/время, а `%2` — имя цели.

* `"QuestAcceptedTitle"` / `"QuestAcceptedText"` — Уведомление при успешном взятии квеста.
*(Пример: "Квест принят" / "Задание %1 добавлено в ваш КПК!")*
* `"QuestCompletedTitle"` / `"QuestCompletedText"` — Уведомление, когда все условия выполнены и квест готов к сдаче.
* `"QuestFailedTitle"` / `"QuestFailedText"` — Сообщение, если квест провален (например, из-за смерти персонажа).
* `"QuestCanceledTitle"` / `"QuestCanceledText"` — Сообщение при ручной отмене задания игроком в меню.
* `"QuestTurnInTitle"` / `"QuestTurnInText"` — Уведомление в момент успешной сдачи НПС и получения награды.
* `"QuestObjectiveCompletedTitle"` / `"QuestObjectiveCompletedText"` — Оповещение о выполнении одной конкретной промежуточной цели внутри квеста.
* `"AchievementCompletedTitle"` / `"AchievementCompletedText"` — Особое уведомление при скрытом выполнении ачивки-достижения.
* `"QuestCooldownTitle"` / `"QuestCooldownText"` — Выводится, если игрок пытается повторно взять циклическое задание, которое еще находится на перезарядке.
* `"QuestNotInGroupTitle"` / `"QuestNotInGroupText"` — Ошибка при попытке взять групповой квест одиночкой.
* `"QuestNotGroupOwnerTitle"` / `"QuestNotGroupOwnerText"` — Ошибка прав, если обычный член отряда пытается сдать квест при строгом режиме `GroupQuestMode`.
