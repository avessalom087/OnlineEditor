
# Документация по настройке DayZ Expansion (Часть 1)

## 1. AI SETTINGS (Настройки ИИ)

| Параметр | Тип данных | Описание |
| :--- | :--- | :--- |
| `"m_Version"` | Integer | Текущая версия настроек. **Никогда не изменяйте это значение**, если только вы точно не понимаете, что делаете. Оно используется системой для автоматического обновления старых конфигураций при выходе обновлений мода. |
| `"AccuracyMin"` | Float | Минимальная точность ИИ (от `0.0` до `1.0`). Может быть переопределена индивидуально для каждого патруля. |
| `"AccuracyMax"` | Float | Максимальная точность ИИ (от `0.0` до `1.0`). Может быть переопределена индивидуально для каждого патруля. |
| `"ThreatDistanceLimit"` | Float | Максимальная дистанция (в метрах), на которой ИИ может рассчитывать потенциальную цель как угрозу. Может быть переопределена индивидуально для каждого патруля. |
| `"NoiseInvestigationDistanceLimit"` | Float | Максимальное расстояние, на котором ИИ услышит шум и решит отправиться на его проверку. Может быть переопределена индивидуально для каждого патруля. |
| `"DamageMultiplier"` | Float | Множитель урона, который наносит ИИ. `1.0` = 100% (базовый урон). Может быть переопределен индивидуально для каждого патруля. |
| `"DamageReceivedMultiplier"` | Float | Множитель урона, который ИИ получает при атаке на него. `1.0` = 100%. Может быть переопределен индивидуально для каждого патруля. |
| `"Admins"` | TStringArray | Список SteamID64 (в кавычках), которым разрешен доступ к меню отладки ИИ (Debug Menu). |
| `"Vaulting"` | Boolean | Разрешить ИИ перешагивать/перепрыгивать препятствия:<br>`0` — ИИ не будет перепрыгивать объекты.<br>`1` — ИИ может перепрыгивать через препятствия. |
| `"SniperProneDistanceThreshold"` | Float | Дистанционный порог для снайперов. Определяет, насколько близко может подойти цель, прежде чем ИИ с продольно-скользящей винтовкой (болтовкой) ляжет на землю. Если цель ближе этого значения, ИИ ложиться не будет. |
| `"Manners"` | Boolean | Поведение и манеры ИИ после боя:<br>`0` — ИИ не будет использовать насмешки или эмоции.<br>`1` — ИИ будет случайно насмехаться или показывать эмоции после убийства цели. |
| `"MemeLevel"` | Integer | "Мемный" уровень. Заставляет ИИ вести себя глупо или забавно:<br>`0` — выключено.<br>`???` — делает что-то забавное. |
| `"CanRecruitFriendly"` | Boolean | Возможность вербовки дружественного ИИ:<br>`0` — дружественных ИИ нельзя нанять в отряд.<br>`1` — игроки могут нанимать дружественных ИИ (в зависимости от отношений между фракциями). |
| `"CanRecruitGuards"` | Boolean | Возможность вербовки охранников:<br>`0` — охранников (Guards) нельзя нанять в отряд.<br>`1` — игроки могут нанимать охранников. |
| `"PreventClimb"` | TStringArray | Список класснеймов (классов объектов) в кавычках, на которые ИИ не должен забираться. Например, `"Land_Mil_Airfield_HQ"` (этот зашит в код по умолчанию). |
| `"PlayerFactions"` | TStringArray | Список фракций в кавычках. Одна из указанных фракций будет случайным образом присвоена новым игрокам при первом заходе. |
| `"LogAIHitBy"` | Boolean | Включить логирование попаданий по ИИ в админ-лог (`true`/`false`). |
| `"LogAIKilled"` | Boolean | Включить логирование убийств ИИ в админ-лог (`true`/`false`). |
| `"EnableZombieVehicleAttackHandler"` | Boolean | Если включено, зомби будут атаковать автомобили, если внутри сидит игрок (`true`/`false`). |
| `"EnableZombieVehicleAttackPhysics"` | Boolean | Если включено, зомби будут раскачивать и физически трясти машину во время атаки (`true`/`false`). |
| `"LightingConfigMinNightVisibilityMeters"` | Object | Настройка того, как далеко ИИ видит ночью без фонарика или ПНВ. Зависит от значения параметра `lightingConfig` в файле `CfgGameplay.json`. |

### Пример настройки ночной видимости по умолчанию (для Чернаруси):
```json
"LightingConfigMinNightVisibilityMeters": {
    "0": 100.0, // При значении lightingConfig 0, видимость 100 метров
    "1": 10.0   // При значении lightingConfig 1, видимость 10 метров
}

```

## 2. AI PATROL SETTINGS (Настройки патрулей ИИ)

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"m_Version"` | Integer | Текущая версия настроек. **Никогда не изменяйте это значение**, если только вы точно не понимаете, что делаете. Оно используется для автоматической конвертации старых настроек при обновлениях мода. |
| `"Enabled"` | Boolean | Включение/выключение патрулей:<br>

<br>`0` — ВЫКЛ |
| `"FormationScale"` | Float | Масштаб построения. Определяет, насколько далеко друг от друга находятся бойцы в формации.<br>

<br>• Для построений по сетке (таких как *Vee, InvVee, Wall, Column* и *InvColumn*) значение задает длину стороны сетки в метрах.<br>

<br>• Для построений не по сетке (*Star, StarDot, Circle* и *CircleDot*) значение примерно определяет расстояние между позициями (в звездных формациях — между внутренними позициями, а внешние масштабируются пропорционально).<br>

<br>**По умолчанию:** `-1.0` (использует значение из `AISettings.json`, которое по умолчанию равно `1.0`). |
| `"DespawnTime"` | Float | Время (в секундах), через которое патруль деспавнется (исчезнет), если в радиусе деспавна (`DespawnRadius`) нет игроков.<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"RespawnTime"` | Float | Время (в секундах), по истечении которого данный патруль может возродиться (отреспавниться), если в радиусе `MinDistRadius` нет игроков.<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"MinDistRadius"` | Float | Минимально необходимое расстояние от игрока для спавна. Если игрок находится ближе этого значения (в метрах), патруль не появится.<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"MaxDistRadius"` | Float | Максимально допустимое расстояние от игрока для спавна. Если игрок находится дальше этого значения (в метрах), патруль не появится.<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"DespawnRadius"` | Float | Радиус деспавна относительно игрока. Если игрок находится ближе этого расстояния (в метрах), патруль **не** деспавнится.<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"AccuracyMin"` | Float | Минимальная точность ИИ для этого патруля (от `0.0` до `1.0`).<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"AccuracyMax"` | Float | Максимальная точность ИИ для этого патруля (от `0.0` до `1.0`).<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"ThreatDistanceLimit"` | Float | Расстояние (в метрах), на котором цель начинает считаться потенциальной угрозой. ИИ не увидит врагов дальше этого расстояния, так что используйте с осторожностью.<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"NoiseInvestigationDistanceLimit"` | Float | Максимальное расстояние (в метрах), на котором ИИ будет искать источники шума.<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"MaxFlankingDistance"` | Float | Максимальное расстояние до врага (в метрах), на котором ИИ будет пытаться обойти его с фланга (**По умолчанию:** `200`).<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"EnableFlankingOutsideCombat"` | Integer | Включить фланкирование вне боя. Обратите внимание, что это дает ИИ своего рода «шестое чувство» в отношении врагов, но не обязательно их точное местоположение при отсутствии прямой видимости.<br>

<br>`0` — отключено.<br>

<br>`1` — включено.<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"DamageMultiplier"` | Float | Множитель наносимого урона от ИИ (**По умолчанию:** `1.0` = без изменений). Базовый урон, который они наносят, умножается на это значение.<br>

<br>`-1` — использовать значение из `AISettings.json`. |
| `"DamageReceivedMultiplier"` | Float | Множитель получаемого ИИ урона (**По умолчанию:** `1.0` = без изменений). Базовый урон, который они получают, умножается на это значение.<br>

<br>`-1` — использовать значение из `AISettings.json`. |

## 3. [Server Hosting] AI Load Balancing (Балансировка нагрузки ИИ)

Файл `mpmissions/dayzoffline.mapname/expansion/settings/AIPatrolSettings.json` содержит настройки балансировки нагрузки.

### Назначение `LoadBalancingCategories`

Этот параметр позволяет указать, скольким патрулям каждого типа разрешено спавниться или оставаться на карте в зависимости от количества игроков онлайн.

> ⚠️ **Важная механика деспавна:** Если патруль уже заспавнен, но заходит больше игроков, чем позволяет лимит балансировки для текущего онлайна, патруль автоматически деспавнится. При этом ИИ садится на землю (точно так же, как игроки при выходе из игры) и становится бессмертным на время фазы деспавна.

Каждому отдельному патрулю может быть назначена категория балансировки (`LoadBalancingCategory`). Каждая категория может содержать одну или несколько записей, определяющих диапазоны онлайн-игроков и максимальное количество патрулей для каждого диапазона.

Если категория не назначена (`""`), используются следующие значения по умолчанию:

* Патрули объектов (`ObjectPatrols`) будут использовать категорию `ObjectPatrol` (если она существует).
* Обычные патрули (`Patrols`) будут использовать категорию `Patrol` (если она существует).
* Квестовые ИИ (например, `AICamp` или `AIPatrol`, когда загружен мод *DayZ-Expansion-Quests*) будут использовать категорию `Quest` (если она существует).

#### Глобальные лимиты (`Global`)

Если категория `Global` существует, она **всегда используется в дополнение к индивидуальным категориям**. То есть, если индивидуальная категория не ограничивает патруль, глобальный лимит (если он настроен) все равно будет действовать.

> 💡 **Примечание:** Значение `-1` для `MaxPatrols` означает, что лимит для этой конкретной записи отсутствует.
> Вы можете редактировать, добавлять и удалять категории по своему вкусу. Нет необходимости придерживаться дефолтных названий (но учтите, что для глобального ограничения категория обязательно должна называться `Global`). Удаление всех категорий означает, что лимиты на сервере действовать не будут.

### Пример конфигурации по умолчанию:

```json
"LoadBalancingCategories": {
    "Example": [
        { "MinPlayers": 0, "MaxPlayers": 255, "MaxPatrols": -1 }
    ],
    "Quest": [
        { "MinPlayers": 0, "MaxPlayers": 255, "MaxPatrols": -1 }
    ],
    "ObjectPatrol": [
        { "MinPlayers": 0, "MaxPlayers": 255, "MaxPatrols": 5 }
    ],
    "HelicopterWreck": [
        { "MinPlayers": 0, "MaxPlayers": 255, "MaxPatrols": 3 }
    ],
    "Global": [
        { "MinPlayers": 0, "MaxPlayers": 10, "MaxPatrols": 10 },
        { "MinPlayers": 11, "MaxPlayers": 20, "MaxPatrols": 8 },
        { "MinPlayers": 21, "MaxPlayers": 30, "MaxPatrols": 6 },
        { "MinPlayers": 31, "MaxPlayers": 40, "MaxPatrols": 4 },
        { "MinPlayers": 41, "MaxPlayers": 50, "MaxPatrols": 2 },
        { "MinPlayers": 51, "MaxPlayers": 255, "MaxPatrols": 0 }
    ],
    "ContaminatedArea": [
        { "MinPlayers": 0, "MaxPlayers": 255, "MaxPatrols": 2 }
    ],
    "Patrol": [
        { "MinPlayers": 0, "MaxPlayers": 255, "MaxPatrols": 5 }
    ]
}

```

## 4. AI Location Settings (Настройки локаций ИИ)

Этот файл управляет тем, как ИИ перемещается и ориентируется в игровом мире.
Вы можете найти этот файл по пути: `mpmissions\<имя_карты>\expansion\settings\AILocationSettings.json`

### Основные параметры

#### `"RoamingLocations"`

Список доступных локаций для бродячего ИИ (актуально, если в файле `AIPatrolSettings.json` для параметра `"WaypointBehavior"` задано значение `"ROAMING"`).

Этот список заполняется автоматически на основе названий локаций, найденных в конфигах карты (`CfgWorlds <mapname> Names`). Если на используемой карте нет именованных локаций, список случайных точек сгенерируется автоматически через 10 секунд после завершения полной загрузки сервера.

Каждая локация должна иметь следующий формат:

```json
{
    "Name": "Settlement_Capital (Petropavlovsk-Sakhalinsky)",
    "Position": [4749.0, 0.0, 10695.0],
    "Radius": 1000.0,
    "Type": "Capital",
    "Enabled": 1
}

```

* **Name** — название, используемое исключительно для того, чтобы администратору было проще различать локации.
* **Type** — тип локации, который подтягивается из `CfgWorlds`. При добавлении кастомных точек это поле можно оставить пустым.
* **Enabled** — статус активности точки (`1` или `true` — ИИ может выбрать эту локацию; `0` или `false` — локация будет игнорироваться).

#### `"ExcludedRoamingBuildings"`

Список класснеймов зданий или их частей, которые бродячий ИИ (`"WaypointBehavior": "ROAMING"`) **не должен** выбирать в качестве конечной точки своего маршрута. Настоятельно рекомендуется оставлять дефолтный список нетронутым и лишь дополнять его своими позициями при необходимости.

#### `"NoGoAreas"`

Список запретных зон, которые ИИ будет гарантированно избегать. Используйте этот параметр как можно реже, так как обилие подобных зон сильно снижает производительность серверного просчета путей (pathfinding).

Каждая запретная зона настраивается по следующему шаблону:

```json
{
    "Name": "Hashima Exclusion Zone",
    "Position": [558.0, 0.0, 4558.0],
    "Radius": 300.0,
    "Height": 130.0
}

```

```json
{

    "RoamingLocations": [
```

## 5. AirdropSettings (Настройки аирдропов)

Этот файл управляет логикой вызова, полета самолета и содержимым контейнеров с припасами.

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"m_Version"` | Integer | Текущая версия настроек. Используется для автоматического обновления конфигов. |
| `"ServerMarkerOnDropLocation"` | Boolean | Создание маркера на карте:<br>

<br>`0` — Точный маркер места сброса груза создаваться не будет.<br>

<br>`1` — Сервер создаст маркер на карте в точной точке сброса (куда приземлится ящик). |
| `"Server3DMarkerOnDropLocation"` | Boolean | Отображение 3D-маркера в мире (игнорируйте, если маркер выключен):<br>

<br>`0` — Маркер не будет отображаться в 3D.<br>

<br>`1` — Маркер будет виден как на карте, так и в 3D на экране игрока. |
| `"ShowAirdropTypeOnMarker"` | Boolean | Отображение типа груза на маркере (игнорируйте, если маркер выключен):<br>

<br>`0` — В названии маркера не будет указан тип аирдропа.<br>

<br>`1` — Название маркера будет уточнять тип груза (например: Военный, Медицинский, Стройматериалы). |
| `"HeightIsRelativeToGroundLevel"` | Boolean | Расчет высоты полета от уровня земли:<br>

<br>`0` — Самолет летит на фиксированной высоте и не адаптируется под рельеф местности.<br>

<br>`1` — Самолет будет изменять высоту полета в зависимости от возвышенности земли под ним. |
| `"Height"` | Float | Высота (альтитуда) полета самолета в метрах. |
| `"FollowTerrainFraction"` | Float | От `0` до `1`. Точность следования рельефу земли (актуально, только если `HeightIsRelativeToGroundLevel` равен `1`). |
| `"Speed"` | Float | Скорость полета самолета в м/с. На скорость падения самого ящика не влияет. |
| `"Radius"` | Float | Радиус (в метрах) зоны сброса, в пределах которого самолет скинет контейнер. |
| `"InfectedSpawnRadius"` | Float | Радиус (в метрах) вокруг приземлившегося ящика, в котором будут спавниться враждебные сущности (зомби/ИИ). |
| `"InfectedSpawnInterval"` | Float | Время (в миллисекундах) между спавном каждой враждебной сущности (используется для распределения нагрузки). |
| `"ItemCount"` | Float | Количество предметов в ящике. Это общая настройка на случай, если у самого контейнера ниже параметр `ItemCount` настроен на `-1`. Имеет два режима работы (см. ниже). |

> 💡 **Режимы работы `ItemCount`:**
> * Если число **положительное** (например, `25`), в контейнере появится **ровно** 25 предметов.
> * Если число **отрицательное** (например, `-25`), в контейнере появится **случайное** количество предметов в диапазоне **от 1 до 25**.
> 
>

### Массив `"Containers"` (Контейнеры)

Список контейнеров аирдропа, их лута и типов охраняющих их врагов.

* **`"Container"`** (String) — Класснейм ящика. Мод предоставляет 12 различных скинов (например, `ExpansionAirdropContainer_Medical`, `ExpansionAirdropContainer_Military` и т.д.).
* **`"Usage"`** (Integer) — Условия появления контейнера:
* `0` — Миссии и вызовы игроками.
* `1` — Только случайные серверные аирдропы (миссии).
* `2` — Только вызовы игроков через сигнальную ракету.


* **`"Weight"`** (Float) — Вес (вероятность) спавна именно этого типа контейнера.
* **`"InfectedCount"`** (Integer) — Количество врагов, которое заспавнится вокруг этого контейнера.
* **`"SpawnInfectedForPlayerCalledDrops"`** (Boolean) — Спавнить ли охрану для аирдропов, вызванных игроками (`0` — нет, `1` — да).

#### Структура `"Loot"` (Настройка добычи)

* **`Name`**: Точный класснейм предмета.
* **`Attachments`**: Список модулей/аттачментов, с которыми должен заспавниться этот предмет.
* **`Chance`**: Шанс спавна от `0.00` (0%) до `1.00` (100%).
* **`QuantityPercent`**: Процент заполненности/количества предмета (от `0` до `100`). `-1` — дефолт (100% заполнено). `-2` — брать значения из вашего файла `types.xml`.
* **`Max`**: Максимально допустимое количество раз, которое этот предмет может заспавниться в одном контейнере. `-1` — без лимита.
* **`Variants`**: Варианты предметов (другие цвета или обвесы).

#### Пример настройки лута и охраны:

```json
"Loot": [
    {
        "Name": "FirstAidKit",
        "Attachments": [ "BandageDressing" ],
        "Chance": 0.38,
        "QuantityPercent": -1,
        "Max": -1,
        "Variants": [
            { "Name": "FirstAidKit", "Attachments": [ "BandageDressing", "SalineBagIV" ], "Chance": 0.13 }
        ]
    }
],
"Infected": [
    "ZombieBase",
    "eAI_SurvivorM_Mirek|faction:Mercenaries,loadout:BanditLoadout"
]

```

<br>`0` — Игроки не могут скрафтить набор флага.<br>

<br>`0` — Флагшток нужно строить поэтапно (приносить материалы), как в ванильной игре.<br>

<br>`1` — Флагшток строится автоматически и мгновенно сразу после его размещения. |
| `"AutomaticFlagOnCreation"` | Boolean | Автоматическое поднятие флага:<br>

<br>`0` — Игроку нужно вручную найти и повесить флаг на флагшток.<br>

<br>`1` — Случайный флаг будет автоматически добавлен на флагшток при установке. |
| `"GetTerritoryFlagKitAfterBuild"` | Boolean | Возврат набора после постройки:<br>

<br>`0` — Игрок теряет набор флага после начала строительства.<br>

<br>`1` — Набор возвращается в инвентарь игрока после возведения первой стадии флагштока. |

<br>`0` — Настроенные зоны являются единственными местами, где разрешено строительство.<br>

<br>`0` — Только на строительные элементы Expansion.<br>

<br>`1` — На элементы Expansion + ванильные заборы.<br>

<br>`2` — На элементы Expansion + ванильные заборы + палатки.<br>

<br>`3` — На элементы Expansion + палатки. |

<br>`-1` — Только участники этой территории могут разобрать флагшток голыми руками.<br>

<br>`0` — Кто угодно может разобрать флагшток голыми руками.<br>

Вместо того чтобы хранить лут как аттачменты или внутри инвентаря ящиков в мире, вещи сохраняются в виртуальную БД сервера и полностью удаляются из игрового мира, когда контейнер закрывается (и восстанавливаются при открытии). Это значительно повышает производительность сервера. |

#### Списки исключений и белые списки предметов:

* **`"DeployableOutsideATerritory"`** — Список класснеймов предметов, которые игрокам разрешено разворачивать на земле вне приватных зон (например, `["Campfire", "BearTrap"]`), если общее строительство вне привата запрещено.

* **`"VirtualStorageExcludedContainers"`** — Список класснеймов контейнеров, которые вы хотите исключить из системы виртуального хранилища.

| `"EnablePartyTab"` | Boolean | Вкладка группы (создание отрядов, управление и инвайты). |
| `"EnableServerInfoTab"` | Boolean | Вкладка информации о сервере (описание проекта, кнопки-ссылки на Discord/сайт). |

## 8. ChatSettings (Настройки чата)

Этот файл управляет доступными каналами текстового чата на сервере и позволяет полностью изменить их цветовую палитру.

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"EnableGlobalChat"` | Boolean | Глобальный чат (синий): возможность общаться со всеми игроками на сервере на любом расстоянии. |
| `"EnablePartyChat"` | Boolean | Чат группы (зеленый): требует наличия мода *DayZ-Expansion-Groups*. Общение внутри отряда. |

#### Цвета каналов чата (`"ChatColors"`)

Задаются в формате **Hex (HEX-код цвета без знака #)**. По умолчанию настроены следующие каналы:

* `"SystemChatColor"` (`"EB45EBFF"`) — Системные сообщения от СЕРВЕРА. Видны всем.
* `"AdminChatColor"` (`"FF392BFF"`) — Сообщения от АДМИНИСТРАТОРА. Видны всем.
* `"GlobalChatColor"` (`"58C3FFFF"`) — Глобальный канал общения между игроками.
* `"DirectChatColor"` (`"FFFFFFFF"`) — Локальный чат (Прямой разговор на небольшом расстоянии).
* `"TransportChatColor"` (`"FFCE09FF"`) — Чат внутри техники.
* `"PartyChatColor"` (`"0AFA7AFF"`) — Чат вашей группы (Party).
* `"TransmitterChatColor"` (`"F9FF49FF"`) — Чат городской радиовышки (PA-системы) или раций.

## 9. DamageSystemSettings (Настройки системы урона)

Кастомная система распределения урона от взрывов, работающая в дополнение к стандартной (ванильной) системе DayZ. Обеспечивает стабильный урон по базам.

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"Enabled"` | Boolean | Включение системы урона Expansion (`0` — выключено, `1` — включено). |
| `"CheckForBlockingObjects"` | Boolean | Проверка преград на пути взрывной волны:<br>

<br>`0` — Отключено. Единственным фактором является расстояние от взрыва до цели, стены игнорируются.<br>

<br>`1` — Включено. Система проверяет, блокирует ли какой-либо крупный объект или стена путь от взрыва до цели. |

* **`"ExplosionTargets"`** (Массив строк) — Список класснеймов предметов, которые должны ПОЛУЧАТЬ урон от взрывов по новой логике (по умолчанию `["ExpansionBaseBuilding", "ExpansionSafeBase"]`).
* **`"ExplosiveProjectiles"`** (Объект) — Связка класснеймов взрывных снарядов с их конфигурациями боеприпасов (например, `{"Bullet_40mm_Explosive": "Explosion_40mm_Ammo"}`). Необходима для корректной регистрации урона от гранат М79.

## 10. CoreSettings (Основные настройки ядра)

Этот файл отвечает за фундаментальные параметры производительности сервера и логику работы Центральной Экономики (CE).

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"ServerUpdateRateLimit"` | Integer | Ограничение частоты обновления серверных скриптов в секунду. Значение `0` позволяет игре обрабатывать обновления в штатном ванильном режиме, минимально допустимый лимит — `40`. |
| `"ForceExactCEItemLifetime"` | Boolean | Принудительное точное время жизни предметов:<br>

<br>`0` — Рекомендуемое значение. Центральная Экономика использует штатный каскадный/ступенчатый респавн лута.<br>

<br>`1` — Время жизни всех предметов строго соответствует секундам из `types.xml`. Отключает оптимизацию деспавна, что может сильно просадить FPS сервера. |
| `"EnableInventoryCargoTidy"` | Boolean | Функция автосортировки инвентаря:<br>

<br>`0` — Выключено.<br>

<br>`1` — На стороне клиента появится иконка «навести порядок» для одежды и контейнеров. Позволяет автоматически скомпоновать предметы для экономии места. |

## 11. GarageSettings (Настройки виртуального гаража)

Этот файл управляет системой виртуального гаража Expansion, позволяя игрокам прятать технику в виртуальное хранилище.

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"Enabled"` | Boolean | Включение системы гаража (`0` — выключено, `1` — включено). |
| `"AllowStoringDEVehicles"` | Boolean | Разрешить сохранять динамический транспорт, заспавненный через `events.xml` (`0` — нет, `1` — да). Перед включением важно обнулить лимиты этих машин в спавнере, иначе они будут респавниться бесконечно. |
| `"MaxStorableVehicles"` | Integer | Максимальное количество машин, которое один игрок или территория могут хранить в гараже. |
| `"VehicleSearchRadius"` | Float | Радиус обнаружения техники (в метрах) вокруг точки доступа к гаражу. |
| `"CanStoreWithCargo"` | Boolean | Разрешить сохранять машины вместе с вещами в багажнике (`0` — только пустые, `1` — разрешено). |
| `"UseVirtualStorageForCargo"` | Boolean | Способ сохранения багажа: `0` — вещи переносятся на скрытый объект в мире, `1` — вещи уходят в виртуальную базу данных напрямую с машиной. |
| `"NeedKeyToStore"` | Boolean | Требовать автомобильный ключ, привязанный к машине, для её парковки в гараж (`0` — нет, `1` — да). |
| `"EntityWhitelist"` | Array | Список класснеймов объектов/строений, которые работают как точки доступа к гаражу (например, паркоматы). |

#### Режимы доступа и права к гаражу:

* **`"GarageMode"`** (Режим доступа): `0` = *Territory Mode* (нужно состоять в привате флага для доступа к гаражу на этой точке); `1` = *Personal Mode* (доступно всем везде).
* **`"GarageStoreMode"`** (Режим парковки): `0` = *Personal* (только владелец паркует авто); `1` = *Shared* (все члены территории могут парковать авто друг друга); `2` = *Tyrannical* (только владелец привата и модераторы паркуют авто участников).
* **`"GarageRetrieveMode"`** (Режим забора техники): аналогичные индексы `0`, `1`, `2` для регулирования прав на извлечение машин из гаража территории.

<br>`0` — Ваниль (пули летят точно в перекрестие прицела экрана).<br>

<br>`1` — Стиль DayZ Alpha (пули летят строго по направлению ствола оружия). Необходим для работы лазерных прицелов. |
| `"EnableAutoRun"` | Boolean | Включение функции автобега по хоткеям (`SHIFT+Z` — автоспринт, `CTRL+Z` — автошаг, `Z` — автобег). |
| `"UseDeathScreen"` | Boolean | Олдскульный экран смерти в стиле Expansion с кровавым отпечатком руки (`1`) вместо черного экрана (`0`). |
| `"UseDeathScreenStatistics"` | Boolean | Выводить ли игроку личную статистику за текущую жизнь после его гибели (`0` — нет, `1` — да). |

#### Кастомизация цветов HUD (`"HUDColors"`)

Позволяет изменить цвета индикаторов стамины, температуры тела и основных нотификаторов (еды, воды) в формате шестнадцатеричного **RRGGBBAA** или десятичного **R G B A** кода.

## 13. Hardline Settings (Настройки модуля Hardline)

Этот файл управляет хардкорной системой репутации игроков и рангов редкости предметов (RPG-составляющая).

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"UseReputation"` | Boolean | Включение общей системы начисления очков репутации за убийства существ (`true`/`false`). |
| `"EnableItemRarity"` | Boolean | Включение системы редкости предметов. Отображает цветные ранги лута в подсказках и меню. |
| `"UseItemRarityOnInventoryIcons"` | Boolean | Окрашивать слоты и иконки предметов в инвентаре в цвета их ранга редкости (как в РПГ-играх). |
| `"UseItemRarityForMarketPurchase"` | Boolean | Запретить покупать предмет у трейдера, если у игрока недостаточно очков репутации для этого тира предметов. |
| `"UseItemRarityForMarketSell"` | Boolean | Запретить продавать предмет трейдеру, если репутация игрока ниже ранга редкости этого предмета. |
| `"ReputationLossOnDeath"` | Integer | Количество очков репутации, которое игрок теряет в случае своей гибели. |
| `"MaxReputation"` | Integer | Максимальный лимит очков репутации, который игрок может накопить суммарно. |

#### Справочник ID рангов редкости предметов:

В параметрах `PoorItemRequirement` – `ExoticItemRequirement` выставляются требования в очках репутации. Идентификаторы рангов в JSON:
`1` = Poor, `2` = Common, `3` = Uncommon, `4` = Rare, `5` = Epic, `6` = Legendary, `7` = Mythic, `8` = Exotic, `9` = Quest.

* **`"EntityReputation"`** — Словарь начисления очков за убийства (например, `{"ZombieBase": 10, "PlayerBase": 100}`).
* **`"ItemRarity"`** — Словарь ручной разметки редкости предметов (например, `{"mag_cz550_10rnd": 5}`).

## 14. LogsSettings (Настройки логирования)

Управление записью действий игроков и событий сервера в файлы логов сообщества (`0` — отключить запись, `1` — записывать в лог).

* `"Safezone"` — Фиксация входов и выходов игроков из безопасных зон.
* `"VehicleCarKey"` — Логирование привязки ключей, закрытия и открытия замков машин.
* `"VehicleTowing"` — Фиксация любых действий по буксировке или эвакуации транспорта.
* `"VehicleLockPicking"` — Логирование попыток взломать машины отмычкой.
* `"BaseBuildingRaiding"` — Фиксация нанесения урона элементам баз Expansion или сейфам.
* `"CodeLockRaiding"` — Логирование атак и подбора кодов на замках CodeLock.
* `"Territory"` — Запись создания приватных территорий и инвайтов в клан.
* `"Killfeed"` — Логирование каждой смерти выживших на сервере.
* `"Chat"` — Сохранение абсолютно всех текстовых сообщений из всех каналов чата.
* `"AdminTools"` — Фиксация действий админов с использованием специальных инструментов Expansion.
* `"Market"` / `"ATM"` — Запись торговых транзакций на рынке и любых банковских операций в банкоматах.

## 15. MapSettings (Настройки карты и навигации)

Файл `MapSettings.json` привязан к конкретной миссии и находится по пути: `mpmissions\dayzOffline.<имя_карты>\expansion\settings\`

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"EnableMap"` | Boolean | Использовать фирменную цветную карту Expansion (`1`) вместо стандартной черно-белой ванильной карты (`0`). |
| `"ShowPlayerPosition"` | Integer | Маркер положения игрока на карте:<br>

<br>`0` — Местоположение скрыто.<br>

<br>`1` — На карте отображается точный маркер текущего положения («Вы здесь»).<br>

<br>`2` — Маркер отображается только при наличии компаса в инвентаре. |
| `"CanOpenMapWithKeyBinding"` | Boolean | Разрешить открытие карты по нажатию на хоткей (кнопка **M**), не требуя обязательного физического предмета карты в руках. |
| `"NeedGPSItemForKeyBinding"` | Boolean | Чтобы работало открытие карты на **M** или вызов HUD-мини-карты (удержание **N**), в инвентаре обязан лежать прибор GPS. |
| `"CreateDeathMarker"` | Boolean | Автоматическое создание временного маркера на карте, указывающего на место последней гибели персонажа. |
| `"CanCreate3DMarker"` | Boolean | Разрешить игрокам ставить 3D-маркеры, которые будут видны прямо на экране во время бега. |
| `"EnableHUDGPS"` | Boolean | Включение мини-экрана GPS на основном интерфейсе. Игроки могут удерживать **N** для включения виджета и коротко нажимать **N** для смены его режимов. |
| `"EnableHUDCompass"` | Boolean | Включение верхней навигационной панели (ленты компаса в стиле PUBG/Skyrim) вверху экрана. |
| `"NeedCompassItemForHUDCompass"` | Boolean | Полоса компаса вверху экрана появится только в том случае, если у игрока в инвентаре лежит физический компас. |
| `"PlayerLocationNotifier"` | Boolean | Выводить графическое уведомление в стиле оригинального DayZ Mod с названием города и временем при входе в населенный пункт. |
