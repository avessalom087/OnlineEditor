
# Документация по настройке DayZ Expansion (Часть 2)

## 16. General Market Settings (Общие настройки рынка и банкоматов)

Файл `MarketSettings.json` привязан к конкретной карте и находится по пути: `mpmissions\dayzOffline.<имя_карты>\expansion\settings\`

| Параметр | Тип данных | Описание |
| :--- | :--- | :--- |
| `"m_Version"` | Integer | Текущая версия настроек. **Никогда не изменяйте это значение**, так как оно используется для автоматического обновления конфигураций. |
| `"MarketSystemEnabled"` | Boolean | Включение системы рынка:<br>`0` — Полностью отключает систему Торговцев (Market).<br>`1` — Включает систему Торговцев. |
| `"MaxVehicleDistanceToTrader"` | Float | Максимальное расстояние (в метрах) для транспорта стандартного размера (машины, вертолеты, лодки), чтобы он отображался в меню торговца как доступный для продажи. Также задает максимальное расстояние до зоны спавна купленного транспорта. |
| `"MaxLargeVehicleDistanceToTrader"` | Float | Максимальное расстояние (в метрах) для крупного транспорта (например, авианосцев), необходимое для отображения в меню продажи и для определения зоны спавна. |
| `"LargeVehicles"` | Array | Список класснеймов транспортных средств, которые должны использовать лимит расстояния `MaxLargeVehicleDistanceToTrader` вместо стандартного. |
| `"SellPricePercent"` | Float | Процент от стоимости покупки предмета, за который торговцы будут его выкупать по умолчанию (глобальная настройка). **По умолчанию:** `75.0` (75% от цены покупки). |

---

### Точки спавна транспорта (`*SpawnPositions`)

Массивы координат и направлений (ориентации) для спавна купленной у трейдера техники. Возле автоторговца должна быть настроена как минимум одна точка спавна в радиусе, не превышающем `MaxVehicleDistanceToTrader`.

*   **`"LandSpawnPositions"`** — Точки спавна наземного транспорта (машины, грузовики).
*   **`"AirSpawnPositions"`** — Точки спавна воздушного транспорта (вертолеты, самолеты).
*   **`"WaterSpawnPositions"`** — Точки спавна водного транспорта (лодки, катера).

#### Пример настройки точек спавна:
```json
"LandSpawnPositions": [
    {
        "Position": [11903.4, 140.0, 12455.1], // Координаты [X, Y, Z]
        "Orientation": [24.0, 0.0, 0.0]        // Направление [Яо, Тангаж, Крен]
    }
]

```

### Настройки банковской системы и банкоматов (ATM)

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"ATMSystemEnabled"` | Boolean | Включение банкоматов:<br>

<br>`0` — Банкоматы полностью отключены.<br>

<br>`1` — Включает банковские терминалы/банкоматы. |
| `"MaxDepositMoney"` | Integer | Максимальное количество денег, которое игрок может хранить на своем личном банковском счете. |
| `"DefaultDepositMoney"` | Integer | Стартовый баланс. Количество денег, которое автоматически зачислится на банковский счет игрока при его самом первом заходе на сервер. |
| `"ATMPlayerTransferEnabled"` | Boolean | Денежные переводы:<br>

<br>`0` — Переводы денег между игроками через банкомат отключены.<br>

<br>`1` — Игроки могут переводить деньги со своего счета на счета других игроков. |
| `"ATMPartyLockerEnabled"` | Boolean | Банковский счет группы:<br>

<br>`0` — Общий счет отряда отключен.<br>

<br>`1` — Включает общий банковский счет для зарегистрированных групп/кланов (Party). |
| `"MaxPartyDepositMoney"` | Integer | Максимальное количество денег, которое группа может суммарно хранить на своем общем счете отряда. |

#### `"Currencies"` (Массив строк)

Список класснеймов предметов, которые банкомат принимает в качестве физической валюты для зачисления на электронный счет игрока.

```json
"Currencies": [
    "expansiongoldbar",
    "expansiongoldnugget"
]

```

#### `"MarketMenuColors"` (Кастомизация цветов рынка)

Позволяет изменить цвета элементов интерфейса рынка (кнопки купить/продать, фоны панелей, тултипы и иконки) в формате шестнадцатеричного **RRGGBBAA** или десятичного **R G B A** кода.

* **`"CurrencyIcon"`** (String) — Путь к `.edds` текстуре иконки валюты, которая будет отображаться в интерфейсе рынка. По умолчанию: `"DayZExpansion/Market/GUI/icons/coinstack2_64x64.edds"`.

## 17. Market Categories Settings (Настройки категорий рынка)

Каждая категория товаров на рынке хранится в отдельном `.json` файле внутри папки категорий. Этот файл задает внешний вид категории в меню, базовые объемы запасов (динамический склад) и свойства входящих в неё предметов.

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"m_Version"` | Integer | Текущая версия настроек. Используется для автоматического обновления конфигураций. |
| `"DisplayName"` | String | Отображаемое название категории в меню рынка. Поддерживает как обычный текст (`"Ammo Boxes"`), так и ключи локализации (`"#STR_EXPANSION_MARKET_CATEGORY_AMMOBOXES"`). |
| `"Icon"` | String | Иконка категории. Задается текстовым именем из списка дефолтных иконок Expansion (например, `"Deliver"`). |
| `"Color"` | String | HEX-код цвета для этой категории в формате **RRGGBBAA** (без знака #). Например: `"FBFCFEFF"`. |
| `"IsExchange"` | Boolean | Обменный пункт (валютная категория):<br>

<br>`0` — Обычные товары.<br>

<br>`1` — Все предметы в данном файле помечаются как **валюта**. Используйте это значение **только** для файлов, содержащих исключительно деньги, слитки или монеты. |
| `"InitStockPercent"` | Float | Процент от максимального запаса (`MaxStockThreshold`), которым будет инициализироваться каждый предмет в торговых зонах при самом первом запуске сервера. **По умолчанию:** `75.0` (75% склада заполнено). |

---

### Настройка предметов внутри категории (`"Items"`):

* **`"ClassName"`** (String) — Точный класснейм предмета.
* **`"MaxPriceThreshold"`** (Integer) — Максимальная цена предмета (когда его запас на складе равен **минимуму**).
* **`"MinPriceThreshold"`** (Integer) — Минимальная цена предмета (когда его запас на складе равен **максимуму**).
* **`"SellPricePercent"`** (Float) — Индивидуальный процент цены продажи. Значение `-1.0` означает использование глобальных настроек или настроек конкретной зоны.
* **`"MaxStockThreshold"`** (Integer) — Максимально допустимое количество этого товара на складе рынка.
* **`"MinStockThreshold"`** (Integer) — Минимально допустимое количество товара на складе.
* **`"QuantityPercent"`** (Integer) — Процент заполненности/ёмкости заспавненного предмета (для магазинов, жидкостей, патронов). Значение `-1` устанавливает 100% заполненность.
* **`"SpawnAttachments"`** (Array) — Список класснеймов модулей/обвесов, которые автоматически установятся на предмет при покупке.
* **`"Variants"`** (Array) — Список класснеймов-вариантов (например, разные цвета рюкзака или одежды). Они будут отображаться в выпадающем меню внутри карточки базового товара.

> 💡 **Полезные лайфхаки для экономики:**
> * **Бесконечный (статичный) склад:** Установите параметры `MinStockThreshold` и `MaxStockThreshold` в значение `1`.
> * **Фиксированная (статичная) цена:** Установите параметры `MinPriceThreshold` и `MaxPriceThreshold` на одинаковую сумму.
> 
> 

#### Пример настройки предметов:

```json
"Items": [
    {
        "ClassName": "childbag_red",
        "MaxPriceThreshold": 40,
        "MinPriceThreshold": 20,
        "SellPricePercent": -1.0,
        "MaxStockThreshold": 100,
        "MinStockThreshold": 1,
        "QuantityPercent": -1,
        "SpawnAttachments": [],
        "Variants": [ "childbag_blue", "childbag_green" ]
    }
]

```

## 18. Trader Settings (Настройки Торговцев)

Этот файл конфигурирует ассортимент конкретного НПС-торговца.

> ⚠️ **Важное правило:** НПС будут работать как торговцы только в том случае, если они физически находятся внутри радиуса действия торговой зоны. См. *Market Zones Settings*.

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"DisplayName"` | String | Имя торговца, отображаемое в верхней части меню (поддерживает обычный текст или `#STR_` локализацию). |
| `"TraderIcon"` | String | Иконка торговца (например, `"Gas"` для продавца запчастей/топлива). |
| `"MinRequiredReputation"` | Integer | *Требуется модуль Hardline.* Минимальная репутация, необходимая игроку для открытия меню торговли с этим НПС. |
| `"MaxRequiredReputation"` | Integer | *Требуется модуль Hardline.* Максимальный порог репутации, выше которого торговец перестанет взаимодействовать с игроком. |

#### `"Currencies"` (Валюта торговца)

Массив класснеймов валют, которые принимает данный конкретный торговец. Сама валюта должна быть предварительно создана в папке Market.

```json
"Currencies": [ "expansionbanknoteeuro" ]

```

---

### Привязка категорий и предметов к торговцу

Доступность товаров регулируется цифровыми индексами (`0`, `1`, `2`, `3`):

* **`0`** — Товар можно **только купить** у этого торговца (торговец его не выкупает).
* **`1`** — Товар можно **и покупать, и продавать** (полный оборот).
* **`2`** — Товар можно **только продать** этому торговцу (в магазине его нет, но НПС его принимает).
* **`3`** — Товар **скрыт из общего списка**, но доступен для кастомизации других вещей (как встроенный обвес для оружия или запчасть для автомобиля).

#### `"Categories"` (Подключение целых категорий)

Используются имена файлов категорий (без расширения `.json`). Чтобы задать режим торговли для всей категории, используйте синтаксис `"ИмяФайла:Индекс"`.

```json
"Categories": [ "Cars:1", "Vehicle_Parts:3" ]

```

#### `"Items"` (Поштучное добавление предметов)

Словарь предметов, где ключ — класснейм, а значение — индекс прав доступа (`0-3`).

```json
"Items": {
    "expansioncarkey": 0,
    "engineoil": 2
}

```

## 19. Market TraderZones Settings (Настройки торговых зон рынка)

В отличие от большинства других параметров, файлы торговых зон находятся в папке: `mpmissions\dayzOffline.<имя_карты>\expansion\traderzones\`

> ⚠️ **Важное примечание:** Торговые зоны представляют собой сферы (3D-объекты), поэтому высоту (вторая координата вектора) для параметра `Position` необходимо настраивать очень точно, иначе торговец на возвышенности или в низине может оказаться вне зоны.

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"m_DisplayName"` | String | Отображаемое имя торговой зоны. Можно указать любое название, оно используется для логов сервера. |
| `"Position"` | Vector | Координаты центра торговой зоны в игровом мире `[X, Y, Z]`. |
| `"Radius"` | Float | Радиус действия торговой зоны в метрах. |
| `"BuyPricePercent"` | Float | Наценка на покупку. Управляет ценой покупки для всех предметов в этой конкретной зоне. По умолчанию равно `100.0`. |
| `"SellPricePercent"` | Float | Коэффициент продажи. Управляет ценой, за которую торговцы этой зоны выкупают вещи у игроков. По умолчанию равно `-1.0`, что означает использование глобального значения из `MarketSettings.json`. |

#### `"Stock"` (Локальный склад зоны)

Объект формата `Map<String, Integer>`, который содержит список предметов, доступных для покупки в этой конкретной зоне, и их **текущее точное количество на складе**.

> 💡 **Динамический склад:** Если вы установите количество предмета на `0`, торговцы в этой зоне вообще не будут иметь его в наличии. Они начнут продавать этот предмет другим выжившим только после того, как кто-то из игроков сам найдет и продаст хотя бы один экземпляр этого товара торговцу.

## 20. MissionSettings (Настройки миссий / Событий)

Этот файл отвечает за автоматический запуск сервером периодических динамических миссий (ивентов).

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"Enabled"` | Boolean | Включение системы миссий:<br>

<br>`0` — Все автоматические миссии отключены.<br>

<br>`1` — Система миссий активна. *(На текущий момент доступны миссии с аирдропами)*. |
| `"InitialMissionStartDelay"` | Integer | Задержка запуска первой миссии после рестарта сервера (в **миллисекундах**). |
| `"TimeBetweenMissions"` | Integer | Интервал времени между завершением старой и стартом новой миссии (в **миллисекундах**). |
| `"MinMissions"` | Integer | Минимальное количество одновременно активных миссий на карте. Число не должно превышать значение параметра `MaxMissions`. |
| `"MaxMissions"` | Integer | Максимальное количество одновременно активных миссий на карте. |
| `"MinPlayersToStartMissions"` | Integer | Минимально необходимое количество игроков онлайн на сервере для запуска миссий. Если онлайн меньше этого числа, ивенты запускаться не будут. |

## 21. NameTagsSettings (Настройки именных тегов / Никнеймов)

Этот файл управляет отображением имен игроков (никнеймов) и значков над головами персонажей при сближении или наведении прицела.

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"EnablePlayerTags"` | Boolean | Включение тегов игроков:<br>

<br>`0` — Никнеймы над головами полностью отключены.<br>

<br>`1` — Никнеймы включены. Если посмотреть на игрока, отобразится его имя. |

* **`"PlayerTagViewRange"`** (Integer) — Максимальное расстояние (в метрах), на котором становится виден никнейм при взгляде на другого игрока.
* **`"PlayerTagsIcon"`** (String) — Путь к файлу иконки формата `.edds`. Позволяет заменить стандартный значок над головой.
* **`"PlayerTagsColor"`** / **`"PlayerNameColor"`** (Integer) — Цвет значка и текста имени игрока. Задается числовым кодом **ARGB int** (конвертируется из RGBA).

#### Ограничение отображения по зонам:

* **`"ShowPlayerTagsInSafeZones"`** (Boolean) — Если установлено `1`, никнеймы игроков будут отображаться **исключительно** внутри безопасных зон (Safezones).
* **`"ShowPlayerTagsInTerritories"`** (Boolean) — Если установлено `1`, никнеймы игроков будут отображаться **исключительно** в границах приватных территорий (баз).

## 22. NotificationSettings (Настройки уведомлений и Киллфида)

Глобальный конфигуратор всплывающих сообщений, системных уведомлений и детального лога убийств (Киллфида) на экране.

### Общие настройки и Вход/Выход

| Параметр | Описание |
| --- | --- |
| `"EnableNotification"` | Общий переключатель уведомлений (`0` — полностью выключить все оповещения, `1` — включить). |
| `"ShowPlayerJoinServer"` / `"ShowPlayerLeftServer"` | Показывать оповещение о том, что игрок подключился/отключился от сервера (`0`/`1`). |
| `"JoinMessageType"` / `"LeftMessageType"` | Тип сообщения о входе/выходе:<br>

<br>`0` — Выводить обычным текстом в текстовый чат (слева внизу).<br>

<br>`1` — Отображать в виде красивого графического уведомления в верхнем левом кону экрана. |
| `"ShowTerritoryNotifications"` | Отображать уведомление на экране, когда игрок заходит на чужую или свою приватную территорию флага (`0`/`1`). |

### Уведомления системы Аирдропов (`0` — скрыть, `1` — показывать):

* `"ShowAirdropStarted"` — Начало серверного ивента аирдропа (вылет самолета).
* `"ShowAirdropClosingOn"` — Самолет приблизился к точке сброса контейнера.
* `"ShowAirdropDropped"` — Контейнер успешно сброшен и летит на парашюте.
* `"ShowAirdropEnded"` — Завершение ивента (ящик залутан или уничтожен зомби).
* `"ShowPlayerAirdropStarted"` / `"ShowPlayerAirdropClosingOn"` / `"ShowPlayerAirdropDropped"` — Аналогичные стадии уведомлений для аирдропов, вызванных лично игроком с помощью сигнальной ракетницы.

### Настройки Киллфида (Лог смертей)

* **`"EnableKillFeed"`** (`0`/`1`) — Общее включение лога смертей на сервере.
* **`"KillFeedMessageType"`** — Место отображения киллфида: `0` — в текстовый чат; `1` — в виде всплывающих окон в верхнем левом углу экрана.

#### Переключатели причин смерти (`0` — скрыть, `1` — показывать в киллфиде):

* `"KillFeedFall"` (падение), `"KillFeedCarHitDriver"` (сбит машиной с водителем), `"KillFeedCarHitNoDriver"` (сбит пустой машиной), `"KillFeedCarCrash"` (смерть водителя при ДТП), `"KillFeedCarCrashCrew"` (смерть пассажира при ДТП).
* `"KillFeedHeliHitDriver"` / `"KillFeedHeliHitNoDriver"` / `"KillFeedHeliCrash"` / `"KillFeedHeliCrashCrew"` — аналогичные триггеры для инцидентов с вертолетами и их экипажем.
* `"KillFeedBarbedWire"` (колючая проволока), `"KillFeedFire"` (сгорел в костре), `"KillFeedWeaponExplosion"` (взрыв гранаты/C4), `"KillFeedDehydration"` (жажда), `"KillFeedStarvation"` (голод), `"KillFeedBleeding"` (кровотечение), `"KillFeedSuicide"` (суицид).
* `"KillFeedWeapon"` (огнестрел), `"KillFeedMeleeWeapon"` (холодное оружие), `"KillFeedBarehands"` (забит кулаками), `"KillFeedInfected"` (убит зомби), `"KillFeedAnimal"` (растерзан диким зверем).
* `"KillFeedKilledUnknown"` / `"KillFeedDiedUnknown"` — смерть от неопределенного или полностью неизвестного источника.
* **`"EnableKillFeedDiscordMsg"`** — Пересылка сообщений об убийствах в Discord-канал проекта. *По умолчанию рекомендуется оставить `0` (отключено) и использовать внешние вебхуки вроде CFTools, так как внутренняя система работает нестабильно.*

## 23. PartySettings (Настройки групп / отрядов)

Этот файл управляет системой отрядов, отображением маркеров над головами тиммейтов, быстрыми метками и HUD-состоянием напарников.

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"EnableParties"` | Boolean | Включение системы групп:<br>

## 24. Personal Storage Settings (Настройки личного хранилища)

| Параметр | Тип данных | Описание |

#### `"ExcludedClassNames"` (Черный список предметов)

Массив строк, содержащий класснеймы предметов, которые **запрещено** класть в личное хранилище.

"ExcludedClassNames": [ "SurvivorBase", "ExpansionMoneyBase" ]

#### `"MenuCategories"` (Настройка категорий меню личного склада)

Позволяет структурировать интерфейс личного хранилища, разбивая вещи по вкладкам (Патроны, Одежда, Оружие и т.д.) с использованием кастомных иконок из файлов интерфейса DayZ (`set: имя_сета image: имя_картинки`). Каждая категория содержит массивы `"Included"` (что входит) и `"Excluded"` (исключения).

## 26. RaidSettings (Настройки рейдинга / Взлома баз)

Отвечает за урон взрывчаткой и пулями по строительным элементам, сейфам и замкам. Напоминание: базовое здоровье стен в Expansion — **30 000 HP**.

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"ExplosionTime"` | Integer | Время (в секундах), которое требуется заряду C4 (`ExpansionSatchel`) до момента детонации после его установки. |
| `"EnableExplosiveWhitelist"` | Boolean | Использование белого списка взрывчатки:<br>

<br>`0` — Любые взрывчатые предметы в игре могут наносить урон базам.<br>

<br>`1` — Только виды взрывов, внесенные в массив `"ExplosiveDamageWhitelist"`, смогут наносить урон постройкам. |
| `"ExplosionDamageMultiplier"` | Float | Множитель взрывного урона по элементам баз Expansion. При значении `50.0` граната (50 урона) нанесет стене `50 * 50 = 2500` урона. Значение `0.0` полностью отключит урон от взрывов. |
| `"ProjectileDamageMultiplier"` | Float | Множитель стрелкового урона (от пуль) по элементам баз Expansion. При дефолтном значении `2.0` пуля (65 урона) нанесет стене `65 * 2 = 130` урона. Значение `0.0` отключит урон от огнестрела по стенам. |
| `"CanRaidSafes"` | Boolean | Рейд сейфов:<br>

<br>`0` — Игроки не могут уничтожать сейфы Expansion.<br>

<br>`1` — Сейфы Expansion можно рейдить взрывчаткой или инструментами. *(Сейфы имеют 15 000 — 10 000 HP)*. |
| `"SafeExplosionDamageMultiplier"` / `"SafeProjectileDamageMultiplier"` | Float | Индивидуальные множители взрывного и стрелкового урона конкретно по сейфам Expansion. |
| `"BaseBuildingRaidMode"` | Integer | Режим рейдинга построек Expansion:<br>

<br>`-1` — Элементы баз вообще нельзя зарейдить.<br>

<br>`0` — Все строительные элементы можно рейдить и уничтожать.<br>

<br>`1` — Урон проходит только по дверям и воротам.<br>

<br>`2` — Урон проходит только по дверям, воротам и окнам. |

#### Рейд объектов и замков ручными инструментами:

* **Сейфы и Колючая проволока:** Параметры `SafeRaidTools` / `BarbedWireRaidTools` (списки инструментов), `*RaidToolTimeSeconds` (время 1 цикла в секундах), `*RaidToolCycles` (количество необходимых циклов взлома) и `*RaidToolDamagePercent` (процент поломки инструмента за весь взлом, где 100 = инструмент полностью сломается).
* **`"CanRaidLocksOnWalls"`** (Integer) — Разрешить взлом замков (CodeLock/ваниль) на стенах баз: `0` — нельзя; `1` — можно на стенах, дверях и воротах; `2` — только на дверях стен; `3` — только на воротах стен.
* **`"CanRaidLocksOnFences"`** / **`"CanRaidLocksOnTents"`** (Boolean) — Разрешить или запретить спиливание замков на ванильных заборах и палатках соответственно (`0`/`1`).
* **Параметры взлома замков:** `LockRaidTools` (список инструментов), `LockOnWallRaidToolTimeSeconds` / `LockOnFenceRaidToolTimeSeconds` / `LockOnTentRaidToolTimeSeconds` (время цикла для стен, заборов и палаток), `LockRaidToolCycles` (число циклов) и `LockRaidToolDamagePercent` (износ инструмента).

## 27. SafeZoneSettings (Настройки безопасных зон / Сейвзон)

Этот файл отвечает за работу и геометрию Safe-зон, а также за автоматическую очистку брошенного там мусора и брошенных машин.

### Геометрия Сейвзон (Типы зон):

1. **`"CircleZones"` (Сферические зоны):** Ограничивают круглую область во всех направлениях. Настройка: `"Center"` `[X, Y, Z]` и `"Radius"` в метрах.
2. **`"CylinderZones"` (Цилиндрические зоны):** Круглые зоны, но с четким вертикальным лимитом. Настройка: `"Center"`, `"Radius"` и `"Height"` в метрах.
3. **`"PolygonZones"` (Полигональные зоны):** Зоны произвольной формы, очерченные по точкам. Настройка: `"Positions"` (массив векторов координат углов зоны).

### Работа системы и Автоочистка (Удаление мусора)

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"Enabled"` | Boolean | Включение сейвзон на сервере (`0` — отключены, `1` — включены). |
| `"FrameRateCheckSafeZoneInMs"` | Integer | Частота проверки игроков, зомби и машин внутри зон (в мс). `0` — проверка каждый тик. |
| `"DisableVehicleDamageInSafeZone"` | Boolean | Бессмертие техники:<br>

<br>`0` — Машины внутри сейвзоны могут получать урон.<br>

<br>`1` — Транспорт внутри безопасных зон полностью защищен от любого урона. |
| `"EnableForceSZCleanup"` | Boolean | Принудительная очистка лута с земли:<br>

<br>`0` — Брошенные на землю вещи не будут удаляться.<br>

<br>`1` — Любой предмет, выброшенный игроком на землю в сейвзоне, будет принудительно удален по таймеру. |
| `"ItemLifetimeInSafeZone"` | Integer | Время жизни (в секундах) для предметов на земле в сейвзоне, если включен параметр `EnableForceSZCleanup`. |
| `"EnableForceSZCleanupVehicles"` | Boolean | Эвакуация/Удаление брошенных машин:<br>

<br>`0` — Техника, оставленная на трейдере, стоит там бесконечно.<br>

<br>`1` — Забытые или брошенные в сейвзоне машины будут безвозвратно удалены, когда истечет таймер. |
| `"VehicleLifetimeInSafeZone"` | Float | Время (в секундах), по истечении которого брошенный на трейдере автомобиль будет удален системой очистки. |
| `"ForceSZCleanup_ExcludedItems"` | Array | Белый список очистки. Массив класснеймов предметов, которые система принудительной очистки сейвзоны **никогда не будет удалять** с земли. |

## 28. SpawnSettings (Настройки появления игрока)

Этот файл контролирует всё, что связано с возрождением персонажей: стартовую одежду, оружие, предметы инвентаря, меню выбора точек респавна и кулдауны.

### Стартовая одежда (`Starting Clothing`)

* **`"EnableCustomClothing"`** (`0`/`1`) — Включить спавн в одежде, настроенной в массивах ниже вместо ванильной.
* **`"SetRandomHealth"`** (`0`/`1`) — Вся стартовая одежда будет идеально новой (`0`) или со случайным износом (`1`).
* **Слоты одежды (Массивы строк):** `Headgear` (шапки), `Glasses` (очки), `Masks` (маски), `Tops` (куртки), `Vests` (разгрузки), `Gloves` (перчатки), `Pants` (штаны), `Belts` (ремни), `Shoes` (обувь), `Armbands` (повязки), `Backpacks` (рюкзаки). При указании пула вещей выбор идет случайно.

### Меню выбора точек спавна (`Spawn Selection`)

* **`"EnableSpawnSelection"`** (`0`/`1`) — Открывать ли интерактивную карту для самостоятельного выбора региона спавна при возрождении.
* **`"SpawnOnTerritory"`** (`0`/`1`) — Разрешить игрокам респавн прямо внутри радиуса действия своего флага территории.
* **`"SpawnLocations"`** (Массив регионов) — Задает имя региона (`"Name"`), пул возможных координат респавна внутри него (`"Positions"` `[[X,Y,Z], [X,Y,Z]]`) и статус задержки на повторный выбор этой точки (`"UseCooldown"` `0`/`1`).

### Стартовый лут в инвентаре (`Starting Gear`)

* `"EnableStartingGear"` (`0`/`1`) — Включение кастомных предметов инвентаря.
* `"UseUpperGear"` / `"UsePantsGear"` / `"UseBackpackGear"` / `"UseVestGear"` — Разрешить спавн лута внутри соответствующих элементов одежды/рюкзака (`0` — нет, `1` — да).
* `"UsePrimaryWeapon"` / `"UseSecondaryWeapon"` — Выдавать основное/второстепенное оружие на спину персонажа из соответствующих массивов.
* `"ApplyEnergySources"` (`0`/`1`) — Если включено, все стартовые предметы, требующие батареек (рации, фонари), автоматически получат новую батарейку 9V при спавне.
* **Наполнение инвентаря (`*Gear`):** Массивы объектов с указанием количества и обвесов. Пример: `{"ClassName": "FNX45", "Quantity": 1, "Attachments": ["Mag_FNX45_15Rnd"]}`.

### Пресеты (Лоадауты) и кулдауны респавна:

* `"UseLoadouts"` (`0`/`1`) — Использовать готовые файлы-пресеты из папки `Loadouts`. Настраиваются отдельно в массивах `"MaleLoadouts"` и `"FemaleLoadouts"` с указанием шанса выбора от `0.0` до `1.0`.
* `"SpawnHealthValue"` / `"SpawnEnergyValue"` / `"SpawnWaterValue"` — Стартовые показатели здоровья (HP), сытости и гидратации персонажа при появлении в мире.
* `"EnableRespawnCooldowns"` (`0`/`1`) — Активирует блокировку регионов спавна, у которых в списке локаций включен `UseCooldown`. Длительность блокировки задается в секундах параметром `"RespawnCooldown"`.
* `"RespawnUTCTime"` (`0`/`1`) — Считать время кулдауна по локальному времени сервера (`0`) или по международному UTC (`1`).
* `"PunishMultispawn"` (`0`/`1`) — Наказывать игрока дополнительным временем ожидания за частый суицид и спам выбором одной и той же точки.
* `"PunishCooldown"` (Integer) — Штрафное время в секундах, прибавляющееся к основному кулдауну при абузе.
* `"PunishTimeframe"` (Integer) — Временное окно проверки в секундах. Если игрок респавнится в одном месте быстрее этого времени, срабатывает штраф.

## 30. VehicleSettings (Настройки транспорта и авиации)

Полный конфигурационный файл, управляющий физикой, защитой, угоном и кастомизацией всех видов техники в моде.

### Системы ключей, замков и угона

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"VehicleRequireKeyToStart"` | Integer | Требование ключа для запуска двигателя:<br>

<br>`0` — Завести машину можно без ключа.<br>

<br>`1` — Спаренный ключ обязательно должен лежать либо в инвентаре игрока, либо внутри машины.<br>

<br>`2` — Чтобы завести мотор, игрок обязан взять спаренный ключ прямо в руки. |
| `"VehicleRequireAllDoors"` | Boolean | Чтобы запереть машину на замок, на ней должны быть установлены абсолютно все двери, капот и багажник (`0` — не важно, `1` — обязательно). |
| `"VehicleLockedAllowInventoryAccess"` | Boolean | Разрешить любым игрокам лутать багажник машины, даже если она заперта на замок (`0` — запрещено, `1` — разрешено). |
| `"CanChangeLock"` | Boolean | Разрешить смену автомобильного замка (перекодировку под новый ключ) с помощью инструментов, заданных в `"ChangeLockTools"` за время `"ChangeLockTimeSeconds"`. |
| `"CanPickLock"` | Boolean | Разрешить взлом автомобилей отмычками (`0` — запрещено, `1` — разрешено). |
| `"PickLockChancePercent"` | Integer | Шанс на успешный взлом замка машины отмычкой в процентах (от `0` до `100`). |
| `"PickLockTimeSeconds"` | Integer | Сколько секунд длится процесс взлома машины отмычкой. |
| `"PickLockToolDamagePercent"` | Integer | Сколько процентов прочности теряет отмычка из списка `"PickLockTools"` после одной попытки взлома. |

* **`"MasterKeyPairingMode"`** (Integer) — Режимы Мастер-ключа (инструмента привязки новых ключей к замку): `-1` — бесконечный; `0` — выключен; `1` — лимит использований из `"MasterKeyUses"`; `2` — лимит, восстанавливаемый ремкомплектом электроники или станком; `3` — восстанавливается только на станке `KeyGrinder`.

### Повреждения, ДТП и физика транспорта

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `"DisableVehicleDamage"` | Boolean | Божественный режим для техники:<br>

<br>`0` — Машины бьются в штатном режиме.<br>

<br>`1` — Полное бессмертие: машины не ломаются от ударов и полностью пуленепробиваемы. |
| `"VehicleCrewDamageMultiplier"` | Float | Множитель урона по экипажу при ДТП. `0` — урон отключен, `1` — ваниль, выше `1` — повышенный урон (быстрая смерть/нокаут). |
| `"VehicleSpeedDamageMultiplier"` | Float | Множитель повреждений деталей машины в зависимости от скорости в момент удара (`0` — урон деталям отключен). |
| `"VehicleRoadKillDamageMultiplier"` | Float | Множитель урона по пешеходам (игрокам/зомби) при наезде на них автомобилем. |
| `"RevvingOverMaxRPMRuinsEngineInstantly"` | Boolean | Удерживание стрелки тахометра в красной зоне (перегазовка в отсечке) будет быстро ломать двигатель машины (`0` — отключено, `1` — ваниль). |
| `"VehicleDropsRuinedDoors"` | Boolean | Если дверь, капот или багажник уничтожены в ноль, они автоматически отваливаются от машины и падают на землю (`0` — нет, `1` — да). |
| `"ExplodingVehicleDropsAttachments"` | Boolean | При взрыве машины все её съемные элементы и вещи из багажника отваливаются и падают на землю в виде лута (`0` — сгорают внутри, `1` — падают). |
| `"DesyncInvulnerabilityTimeoutSeconds"` | Float | Окно бессмертия в секундах при рассинхронизации. Выдает неуязвимость машине и экипажу, если сервер зафиксировал жесткий сетевой лаг (защита от полетов и багов физики). |
| `"PlacePlayerOnGroundOnReconnectInVehicle"` | Integer | При вылете или рестарте высаживать игрока на землю рядом с машиной: `0` — выкл, `1` — всегда, `2` — только при рестарте сервера. |

### Особенности симуляции вертолетов и авиации

* `"EnableWindAerodynamics"` (`0`/`1`) — Включение симуляции ветра, сносящего вертолет в сторону при пилотировании.
* `"EnableMainRotorDamage"` / `"EnableTailRotorDamage"` (`0`/`1`) — Включение повреждений несущего и хвостового винтов при столкновениях со столбами/деревьями или при заваливании на бок.
* `"EnableHelicopterExplosions"` (`0`/`1`) — Разрешить вертолетам взрываться при падении с высоты или критическом ударе.
* `"PilotlessAutoHoverEngineStopDelaySeconds"` (Float) — Время (в секундах), которое вертолет провисит в режиме автозависания (`Auto-Hover`) без пилота в кресле до того, как движок заглохнет.
* `"RoughLandingVerticalSpeedThreshold"` (Float) — Порог вертикальной скорости (в м/с), при превышении которого посадка считается жесткой и наносится урон шасси. Значение `99.0` полностью отключает этот урон.
* `"DamagedEngineStartupChancePercent"` (Float) — Вероятность (шанс в процентах) завести мотор, имеющий износ 50% и более.

### Механика Буксировки (`Towing`)

* `0` — Буксировка полностью отключена.
* `1` — Включено. Машины могут буксировать другие машины на тросу. Вертолеты получают возможность буксировать абсолютно любые типы техники по воздуху.

### Защитные маскировочные сетки (Чехлы / Covers)

Накрытие машины маскировочной сетью отключает у объекта физику, что **колоссально увеличивает производительность сервера** и убирает лаги транспорта.

* `"EnableVehicleCovers"` (`0`/`1`) — Разрешить игрокам накрывать транспорт маскировочной сеткой.
* `"AllowCoveringDEVehicles"` (`0`/`1`) — Разрешить накрывать чехлом машины, заспавненные Центральной Экономикой (`events.xml`). *Требует обнуления лимитов в спавнере.*
* `"UseVirtualStorageForCoverCargo"` (`0`/`1`) — При накрытии машины чехлом весь лут из её багажника вырезается из мира и уходит в виртуальную базу данных.
* `"VehicleAutoCoverTimeSeconds"` (Float) — Таймер в секундах, через который пустая машина с выключенным двигателем сама автоматически накроется чехлом.
* `"VehicleAutoCoverRequireCamonet"` (`0`/`1`) — Требовать ли для авто-чехла наличие установленной маскировочной сети в слоте машины.

### Индивидуальная конфигурация моделей (`VehiclesConfig`)

Позволяет переопределить настройки зацепа игроков и взлома для конкретных класснеймов машин. Настраивается в массиве объектов:

* `ClassName`: Класснейм автомобиля.
* `CanPlayerAttach` (`0`/`1`) — Разрешить ли игрокам удерживаться/стоять на крыше или в кузове машины во время её движения.
* `LockComplexity` (Float) — Сложность замка. Влияет на шанс взлома отмычкой (выше значение = ниже шанс вскрыть).
